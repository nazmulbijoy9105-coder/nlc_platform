# DEPLOY.md - NLC Platform Deployment Guide

## OPTION A: Render.com (Recommended)

### Step 1: Push to GitHub
```bash
git init && git add . && git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_ORG/nlc-platform.git
git push -u origin main
```

### Step 2: Deploy Blueprint
1. Go to https://dashboard.render.com
2. New > Blueprint > Select your repo
3. Render reads render.yaml and provisions: PostgreSQL, Redis, API, Worker, Beat
4. Click Apply

### Step 3: Set Secrets (Render Dashboard > each service > Environment tab)
| Variable | Where to get it |
|----------|----------------|
| TOTP_ENCRYPTION_KEY | python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())" |
| JWT_SECRET_KEY | openssl rand -hex 64 (or auto-generated by Render) |
| ANTHROPIC_API_KEY | console.anthropic.com |
| AWS_ACCESS_KEY_ID | AWS IAM console |
| AWS_SECRET_ACCESS_KEY | AWS IAM console |
| S3_BUCKET_NAME | Your S3 bucket name |
| EMAIL_FROM | SES-verified email address |
| ALLOWED_ORIGINS | https://your-frontend.com |

### Step 4: Migrate + Seed (Render Dashboard > nlc-api > Shell)
```bash
alembic upgrade head
python scripts/seed_rules.py
python scripts/seed_prompt_templates.py
```

### Step 5: Create First Super Admin (nlc-api Shell)
```bash
python3 -c "
import asyncio
from sqlalchemy.ext.asyncio import AsyncSession
from app.models.database import engine
from app.services.user_service import UserService

async def run():
    async with AsyncSession(engine) as db:
        svc = UserService(db)
        u = await svc.create_user(
            email='admin@neumlexcounsel.com',
            full_name='Super Admin',
            role='SUPER_ADMIN',
            password='CHANGE_ME_NOW',
        )
        await db.commit()
        print('Created:', u.email)
asyncio.run(run())
"
```

### Step 6: Verify
```bash
curl https://YOUR-APP.onrender.com/api/v1/health
curl https://YOUR-APP.onrender.com/api/v1/rules  # Should return 32 rules
```

## OPTION B: Docker Compose on VPS

### Prerequisites
- Ubuntu 22.04, min 2GB RAM / 2 vCPU
- Docker + Docker Compose installed
- Domain pointing to server IP

```bash
git clone https://github.com/YOUR_ORG/nlc-platform.git
cd nlc-platform
cp .env.example .env
# Edit .env with all required values

# For production with nginx (SSL)
make ssl-cert    # self-signed for testing, or use certbot for real SSL
make prod        # docker compose --profile production up -d

# Migrate + seed
docker compose run --rm api alembic upgrade head
docker compose run --rm api python scripts/seed_rules.py
docker compose run --rm api python scripts/seed_prompt_templates.py
```

## AWS S3 Setup

```bash
aws s3 mb s3://nlc-documents --region ap-south-1
aws s3 mb s3://nlc-backups --region ap-south-1
# Block all public access on both buckets
```

IAM policy needed: s3:PutObject, s3:GetObject, s3:ListBucket on both buckets + ses:SendEmail.
