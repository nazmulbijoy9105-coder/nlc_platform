```
Crash happens
        ↓
Fly.io detects no response (30 sec)
        ↓
Auto-restart triggered
        ↓
New process starts (< 30 sec)
        ↓
Health check passes
        ↓
Users see 0 downtime (transparent restart)
        ↓
Slack notification sent (informational)
```

### Scenario: Bad Deployment
```
Bad code deployed
        ↓
GitHub Actions tests should catch it ⚡
        (if not caught...)
        ↓
Health check fails
        ↓
Slack alert immediate
        ↓
Developer runs: git revert HEAD && git push
        ↓
GitHub Actions auto-rolls back
        ↓
Previous good version deployed
        ↓
Fixed in < 5 minutes
```

### Scenario: Database Corrupted
```
Data corruption detected
        ↓
SSH into backend: flyctl ssh console
        ↓
List backups: flyctl postgres backups list
        ↓
Restore from backup: flyctl postgres backups restore backup-X
        ↓
Run migrations: alembic upgrade head
        ↓
Service recovers < 5 minutes
```
